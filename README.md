# Cyber Security Assessment Project

## Overview

This project focuses on conducting a basic security assessment of the OWASP Juice Shop application. The assessment includes website reconnaissance, security header analysis, SSL/TLS configuration review, OWASP Top 10 research, vulnerability identification, and security hardening recommendations.

## Objectives

* Analyze the target application's technology stack.
* Review website structure and functionality.
* Assess security headers and SSL/TLS configuration.
* Research OWASP Top 10 security risks.
* Identify potential security weaknesses.
* Develop security improvement recommendations.
* Create professional security assessment documentation.

## Target Application

**OWASP Juice Shop**

OWASP Juice Shop is an intentionally vulnerable web application designed for security training, awareness, and testing purposes.

## Assessment Activities

### Website Reconnaissance

* Identified technologies used by the application.
* Documented login, registration, search, and form functionality.

### Security Headers Analysis

* Reviewed security headers.
* Identified missing or weak security configurations.
* Provided security recommendations.

### SSL/TLS Analysis

* Evaluated SSL certificate configuration.
* Reviewed supported protocols and cipher suites.
* Documented SSL security posture.

### OWASP Top 10 Research

* Studied major web application security risks.
* Documented vulnerabilities, impacts, and mitigation techniques.

### Vulnerability Assessment

* Identified potential security weaknesses.
* Assessed risk levels and business impact.
* Recommended remediation actions.

### Security Hardening

* Developed a security improvement plan.
* Proposed authentication, SSL, monitoring, and header enhancements.

## Key Findings

* Several security headers were missing or misconfigured.
* SSL/TLS configuration achieved a strong security rating.
* Security hardening opportunities were identified.
* OWASP Top 10 risks remain critical considerations for web application security.

## Conclusion

This project provided practical experience in performing a security assessment using industry-standard tools and methodologies. The findings and recommendations can help improve the overall security posture of web applications and support secure development practices.
