# OWASP Security Guide

## Broken Access Control
Description: Users gain access beyond their permissions.
Example: Normal user accessing admin pages.
Impact: Unauthorized access.
Prevention: Implement access controls.

## Cryptographic Failures
Description: Weak protection of sensitive data.
Example: Storing passwords in plain text.
Impact: Data exposure.
Prevention: Use strong encryption.

## Injection
Description: Malicious input executed as commands.
Example: SQL Injection.
Impact: Data theft.
Prevention: Input validation and parameterized queries.

## Insecure Design
Description: Security weaknesses in application design.
Example: Missing security controls.
Impact: Increased attack surface.
Prevention: Secure design practices.

## Security Misconfiguration
Description: Improper security settings.
Example: Default credentials.
Impact: Unauthorized access.
Prevention: Secure configuration management.

## Vulnerable Components
Description: Use of outdated software.
Example: Old libraries with known vulnerabilities.
Impact: Exploitation by attackers.
Prevention: Regular updates.

## Authentication Failures
Description: Weak login security.
Example: Weak passwords.
Impact: Account compromise.
Prevention: MFA and strong password policies.
